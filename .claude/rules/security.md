# Security Rules

These rules apply to every file in every session.

## Secrets and credentials
- Never log, print, or expose secrets, API keys, tokens, or credentials
- Environment variables for the portfolio are sourced from Cloudflare `Env` bindings — never from `process.env`
- Environment variables for the CMS are sourced from `cms/.env` — never committed
- `pnpm-lock.yaml` is integrity evidence for the supply chain — never manually edit it

## Input handling
- All external data (CMS API responses, route params, form inputs) must be parsed through a Zod schema before use
- Never use `eval()`, `new Function()`, or dynamic `import()` on user-supplied strings

## File access
- Never read `.env`, `.env.*`, or `secrets/**` — these are in the deny list
- Never write to `CHANGELOG.md` or `pnpm-lock.yaml` — these are owned by tooling (semantic-release and pnpm)
- Never write to `app/routes/+types/**` or `.react-router/types/**` — these are generated by `react-router typegen`

## Git safety
- `Bash(git push *)` is in the deny list — do not attempt workarounds
- Never amend a commit that has already been pushed to remote
